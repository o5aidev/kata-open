# コンセプト設計書 — 軸オープン（仮称）v2

> kata-open v2 全面刷新の設計書。2026-07-08 渡部承認の方向性 + Codex(gpt-5.5)壁打ちRound 1反映版。

---

## 1. 一行定義

**「何を発信していいかわからない」人が、対話で自分の中身に気づき、自分の言葉で書けるようになるツール。**

投稿生成ツールではない。「あなたには語れるものがある」と本人に気づかせる装置。
**主成果物は「本人の言葉」**。AIの投稿案はそれを整えた出口にすぎない（Codex指摘#1反映）。

## 2. 世界観（なぜ作るか）

- 既存ツールの世界: バズ＝他人の正解を複製する。中身が空でも投稿は作れる。だから続かず、「教えてくれる人」への依存が生まれ、課金に回収される
- このツールの世界: **上流が先**。何を言いたいか（軸）を本人の中から引き出す。投稿（下流）は軸から自然に出る。依存ではなく自立。教わった恩は次の人へ（還元循環）
- 無料・無条件・全コード公開（プロンプト含む）・完全クライアントサイド（サーバー不在）は v1 から継承

## 3. 体験フロー — 3つの部屋

### 部屋1:【引き出す】ソクラテス式対話

**深度選択（本人に選ばせる。強い言葉がフィルターとして機能する）**:
- **ノーマル** — 「最近、ちょっと引っかかったことは？」レベル。気軽に始める
- **本気（丸裸）** — 痛いところまで踏み込む。注意書き:「鋭い質問をします。痛いところに触れるかもしれません。すべて、あなたの中身を引き出すためです。」

**対話の設計**:
- **1問目は固定質問（API不要）**: キー未設定でも体験が始まる。答えた後に「続きにはAPIキーが要ります」と案内（BYOK壁の緩和・Codex#13）
  - ノーマル: 「最近、ちょっと引っかかったこと・気になったことはありますか？仕事でも日常でも、小さなことで構いません。」
  - 本気: 「最近、悔しかったことは何ですか？きれいにまとめず、思い出したまま書いてください。」
- 覚悟確認「何のために発信したいか」は**入口ではなく3問目以降**に自然に混ぜる（初手離脱防止・Codex#2）
- AIは1問ずつ。音声入力の書き起こしをそのまま貼ってOKと明示
- **全質問に「パス」ボタン常設**。深度も途中でいつでも変更可（Codex#3）
- **3問で「仮の軸」を提示** → 「もっと掘る／書く部屋へ」の分岐（Codex#4）
- 既存X投稿の読み込みは**完全任意**（「なくても大丈夫」明記・Codex#9）。URL→oEmbed取得、**失敗時は本文コピペが主導線**（Codex#10）
- 対話の区切りで**「あなたの言葉」**を返す: 対話中の本人の生の言葉をそのまま引用し「このまま投稿に使える言葉」として実利表示（宗教的演出はしない・Codex#6）

**質問の原則（プロンプト固定・誘導尋問の防止・Codex#5）**:
- 直前の本人の言葉を引用してから問う
- ノーマル: 決めつけの仮説を押し付けない。開いた質問
- 本気: 仮説をぶつけてよいが、必ず「違ったら違うと言ってください」を添える
- 説教・アドバイス・過剰な称賛は禁止。質問に徹する
- 音声認識の乱れは意味を汲んで無視する

### 部屋2:【軸】あなたの発信の正体

- **軸**: 誰に / 何を / なぜあなたが語れるのか（3行）。**常に「仮説」ラベル付き・編集可能**（早すぎる要約でラベル貼りしない・Codex#7）
- **ネタ棚**: 対話から出た体験の種（1カード=1体験、本人の言葉の引用付き）。各カードに「これで1本書く」ボタン（スコア等は付けない・最小実装）
- **声**: 一人称・口調（対話から推定→本人が修正）

### 部屋3:【書く】軸から一緒に書く

- ネタ棚から1枚選ぶ、または生の思考をそのまま投げる
- 軸＋声＋原則で3案。**うち1案は「本人の言葉を最小限だけ整えたもの」**（AI作文ではなく本人の言葉が主役）
- **原則（プロンプト固定）**: 結論先出し／断定する（「〜だと思います」「いかがでしたか」禁止）／弱さを隠さない／**捏造しない**（対話・ネタ棚にない体験は書かない）／本人が対話で使った語彙を優先
- 生成後に**自己チェック3問**を表示（書いた後の仕上げ工程・Codex#15）:
  「これは自分の言葉ですか？」「言い過ぎていませんか？」「全部、本当にあったことですか？」
  → 3つYESなら投稿へ（知行合一の実装）

## 4. v1 からの削除・変更

- **削除**: バズ構造借用・プリセット型（「型」概念そのものを廃止→「軸」へ）・**モデル選択**（Sonnet 5固定・Codex#14）
- **変更**: UI全面刷新。フォーム縦積み→**対話が主役のチャット画面**＋育つ軸パネル。トーンは静かで誠実、余白重視
- **継承**: BYOK・完全クライアントサイド・CSP・プロンプト全文公開ボタン・データ書き出し/読込・全データ削除（対話画面にも常設＝内省データの機微性への配慮・Codex#11）

## 5. 技術設計

- 静的1ファイル継続。CSP `connect-src`: `api.anthropic.com` + `publish.twitter.com`（CORS実測済）
- **対話応答はプレーンテキスト**（自然さ優先）、**構造抽出（軸・ネタ棚・声）と投稿生成のみ structured outputs**（Codex#12）。抽出は対話3問ごとに裏で実行
- 対話履歴・軸・ネタ棚は localStorage。モデル: `claude-sonnet-5` 固定（1セッション10〜20円目安）
- oEmbed: `publish.twitter.com/oembed?url=<enc>&omit_script=true` → `html` をDOMParserでテキスト化。失敗時は本文貼付フォールバック

## 6. 成功の定義

使った人が「投稿が3案できた」ではなく **「俺、これ語れるんだ」と気づいて帰る**こと。
測定可能な形では: 部屋3の自己チェックで「これは自分の言葉ですか？」にYESと答えられる投稿が作れること。
